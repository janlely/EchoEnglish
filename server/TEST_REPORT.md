## 测试架构

### 测试配置
- **测试框架**: Jest
- **数据库**: SQLite (文件数据库 `test.db`)
- **清理策略**: 每个测试前清理数据库
- **数据隔离**: 使用唯一邮箱避免冲突

### 测试文件结构
```
tests/
├── jest-setup.ts              # Jest 全局设置
├── jest-global-setup.ts       # 数据库迁移
├── jest-global-teardown.ts    # 清理数据库文件
├── auth.service.test.ts       # 认证服务测试
├── chat.service.test.ts       # 聊天服务测试
└── message.service.test.ts    # 消息服务测试
```

### 测试数据管理
1. **全局 Setup**: 运行数据库迁移

   - 清理所有测试数据
   - 创建测试所需的基础数据（用户、聊天等）
3. **Global Teardown**: 删除测试数据库文件
